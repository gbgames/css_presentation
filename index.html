HTML: The Foundation

HTML is semantic.

HTML is text annotated with meaning. For example:

I used <h1> here to say this is a heading

I used <em> here to emphasize the word "here"

I used <a href> here to create a link to my website

HTML: The Foundation

HTML has two types of elements. (Well, not anymore, but it still useful to say so)

Block: structure your page into sections (could contain more block or inline elements)

CSS: The Interior Decorator

If HTML is marked-up content, then CSS is the presentation of that content.

CSS styles allow you to bring out the meaning of HTML. For example:

I used <h1> here to say this is a heading

is styled with:

h1 { box-shadow: 10px 5px 5px 1px rgba(5, 5, 5, 5); background-color: green; display: flex; justify-content: flex-end; font-family: serif; font-weight: bold; font-size: 46px; padding: 10px; }

It could just as easily have been styled with:

h1 { font-family: sans-serif; font-weight: lighter; font-size: 12px; color: red; }

And it would look like this, which isn't very heading-ish, is it?

The Box Model

The margin surrounds the box. It says "Don't let other boxes come too close to me"

Margins don't add up between boxes. They "collapse" instead.

I imagine not knowing about the collapse can result in a Peter-Griffin-wrestling-with-blinds situation.

Putting It All Together

We have a good mental model for how web pages render now:

  • We know about the flow of HTML.

Helping a Friend

A colleague reached out to me asking for help with a non-profit's website.

The staff photos looked distorted.

Had he asked just a few weeks prior, I would not have known how to fix the images.

A quick check, and I changed:

.maincontent img{ height:100px; width:100px; }

to

.maincontent img{ width:100px; }

Nothing major, but BAM! I'm a web developer now, right?
